# 優先度管理UIの改善計画

> **ステータス: 実装済み** (2026-02-26)

## Context

現在のシステムでは外勤先と医員の関係を6つの概念で管理している：
- 外勤先マスタ: `fixed_doctors`(固定), `excluded_doctors`(除外), `preferred_doctors`(指名)
- 優先度マスタ: ◎(月1回必須=2.0), ○(割当可能=1.0), ×(割当不可=0.0)

これらは意味的に重複しており（◎≈固定, ×≈除外, 指名≈ソフトな◎）、
UIが分散してわかりにくい。

**ゴール**: 4ラベル（固定/指名/任意/除外）に統合し、
編集可能なマトリクスで一元管理できるようにする。

## ページ構成（変更後）

```
┌─────────────────────────────────────────────────┐
│ セクション1: 医員・外勤先の基本設定              │ ← 既存（回数上限ボタン削除のみ）
│   医員一覧 | 外勤先一覧                          │
├─────────────────────────────────────────────────┤
│ セクション2: 外勤先の指名・優先度設定            │ ← 全面改修
│   A. 優先度マトリクス（編集可能、4ラベル）        │
│   B. 確認ビュー（固定/指名/除外がある外勤先のみ） │
├─────────────────────────────────────────────────┤
│ セクション3: 医員の希望設定                      │ ← 再構成
│   A. 月回数上限の一括設定                         │
│   B. 日別外勤先希望（既存移動）                   │
│   C. 外勤先の日別設定（既存移動）                 │
└─────────────────────────────────────────────────┘
```

## 概念統合: 6概念 → 4ラベル

| 新ラベル | weight | 意味 | オプティマイザでの扱い |
|---------|--------|------|---------------------|
| **固定** | 3.0 | この外勤先は固定メンバーでまわす | ハード: WL（固定のみ割当可）+ 月1回以上必須 |
| **指名** | 2.0 | できれば来てほしい | ソフト: 割当ボーナス |
| **任意** | 1.0 | デフォルト（制約なし） | ソフト: 通常の割当対象 |
| **除外** | 0.0 | この医員は来ないでほしい / 行かない | ハード: 割当不可 |

**データソースの一元化:**
- 優先度マスタ（doctor_id, clinic_id, weight）が唯一のソース
- 外勤先マスタの `fixed_doctors`, `excluded_doctors`, `preferred_doctors` カラムはオプティマイザから参照しない

## 変更内容

### 1. admin_master.py: セクション2の全面改修

#### 2A. 優先度マトリクス（編集可能）

`st.data_editor` で全医員×全外勤先のマトリクスを表示・編集。

```
             | 外勤先A | 外勤先B | 外勤先C | ...
医員1(フェロー)  | 固定    | 除外    | 任意    |
医員2(大学院生)  | 指名    | 任意    | 除外    |
医員3(レジデント) | 任意    | 任意    | 任意    |
```

- 行: アクティブ医員（job_rank降順 → 名前順。上級医が上）
- 列: アクティブ外勤先
- セル: `SelectboxColumn` で 固定/指名/任意/除外 を選択
- 一括保存 → `batch_set_affinities()` で反映

#### 2B. 確認ビュー（マトリクス下部）

固定・指名・除外が1件以上ある外勤先のみ表示。管理者が設定を目視確認できる。

```
外勤先A:
  固定メンバー: 医員1, 医員4（この外勤先はこのメンバーのみ割り当て）
  除外: 医員2

外勤先C:
  指名: 医員3（できれば割り当て）
  除外: 医員2, 医員5
```

※ 読み取り専用。編集はマトリクス(2A)で行う。

### 2. admin_master.py: セクション3の再構成

#### 3A. 月回数上限の一括設定

医員編集セクションから移動。全医員の上限を一覧で確認・一括編集。

```
医員名         | 役職       | 現在の上限  | 新しい上限
医員1(フェロー)  | フェロー    | 2回/月     | [number_input]
医員2          | 大学院生    | 制限なし    | [number_input]
```

- 保存: `batch_update_max_assignments()` で一括反映
- 医員編集セクションの「回数上限」ボタンとフォームは削除

#### 3B, 3C: 日別設定（既存の移動）

既存の「日別外勤先希望」と「外勤先の日別設定」をセクション3に移動。内容はそのまま。

### 3. admin_master.py: セクション1の微修正

- 医員編集から「回数上限」ボタンとフォームを削除（セクション3Aに移動のため）
- 外勤先編集は変更なし
  - `fixed_doctors`, `excluded_doctors`, `preferred_doctors` カラムは外勤先マスタに残るが、UIからの編集対象外とする（マトリクスが唯一の編集手段）

### 4. optimizer.py: 優先度マスタからの制約導出

#### 定数の変更
```python
PRIORITY_FIXED = 3.0     # 固定: 固定メンバー（WL + 月1回以上）
PRIORITY_NOMINATED = 2.0 # 指名: できれば来てほしい
PRIORITY_ANY = 1.0       # 任意: デフォルト
PRIORITY_EXCLUDED = 0.0  # 除外: 割当不可
```

#### 制約マップ構築
```python
# affinities のみから全制約を導出
clinic_fixed = {}     # {clinic_id: set(doctor_ids)}  weight=3.0
clinic_nominated = {} # {clinic_id: set(doctor_ids)}  weight=2.0
clinic_excluded = {}  # {clinic_id: set(doctor_ids)}  weight=0.0
must_pairs = {}       # {doctor_id: [clinic_ids]}     weight=3.0
never_pairs = {}      # {doctor_id: [clinic_ids]}     weight=0.0
```

#### 目的関数の簡略化
- `nomination_term` を削除（指名の効果は `priority_term` の weight=2.0 で自然に反映）
- `w_nom` パラメータを削除
- モード別重みを7項に簡略化（w_var, w_pref, w_pri, w_avoid, w_cnt, w_dcr, w_suit）

### 5. ml_adjuster.py: フォールバックスコアの調整

```python
# weight範囲の変更: /2.0 → /3.0
adjusted = base_score * (weight / 3.0) if weight > 0 else 0.0
```

### 6. database/master.py: バッチAPI追加

- `batch_set_affinities()`: API呼び出し max 4回
- `batch_update_max_assignments()`: API呼び出し max 4回

### 7. database/__init__.py: エクスポート追加
