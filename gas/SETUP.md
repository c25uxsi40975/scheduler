# メールリマインダー・通知 セットアップ手順

## 機能概要

### 土曜外勤
1. **毎週金曜18時のリマインダー**: 翌日（土曜）の外勤がある医員にメール送信
2. **スケジュール確定通知**: 管理者がスケジュールを確定したタイミングで全医員にメール送信
3. **希望入力確認メール**: 医員が希望を保存するたびに本人へ入力内容の確認メールを送信
4. **全員入力完了通知**: 全医員の希望入力が完了した時点で管理者にメール送信
5. **入力期限リマインダー**: 期限日当日に全医員へ期限通知、翌日に管理者へ未入力者リストを送信

### 平日外勤
6. **平日スケジュール確定通知**: 副管理者がスケジュールを確定したタイミングでセクションメンバーにメール送信
7. **平日希望入力確認メール**: 医員が希望を保存するたびに本人へ確認メールを送信
8. **平日全員入力完了通知**: セクション全員の希望入力が完了した時点で副管理者にメール送信
9. **平日入力期限チェック**: 期限日当日にセクションメンバーへ通知、翌日に副管理者へ未入力者リスト
10. **平日前日リマインダー**: 翌日に平日外勤がある医員に前日通知

## 前提

- 外勤調整データのGoogleスプレッドシートが稼働中であること
- 管理画面で医員のメールアドレスが登録済みであること

## アーキテクチャ

GASは**集中型**（1つのGASプロジェクトで土曜＋全平日セクションを処理）。

```
マスタSS (1つ・共有)
├── 医員マスタ, 外勤先マスタ, 優先度マスタ, 設定
├── 平日外勤設定 ← spreadsheet_key で各セクションSSを参照
├── 平日スロットマスタ
└── スケジュール対象日

土曜運用SS (1つ) ← GAS配置先
├── 希望_YYYY-MM
├── スケジュール_YYYY-MM
└── GAS: common.gs + saturday.gs + weekday.gs

平日セクション別SS (N個) ← GASデプロイ不要
├── 平日希望_{section}
├── 平日スケジュール_YYYY-MM
└── シフト交換_YYYY-MM
```

GASは平日セクション別スプレッドシートに `SpreadsheetApp.openById()` でアクセスするため、
**セクション追加時にGASの変更やデプロイは不要**。

## ステップ1: 管理画面でメールアドレスを登録

1. Webアプリの管理者画面 → マスタ管理タブを開く
2. 各医員の「📧設定」ボタンからメールアドレスを登録する
3. メールアドレス未設定の医員にはリマインダー・通知は送信されない

## ステップ2: Google Apps Scriptを設定

1. **土曜運用データ用スプレッドシート**（月別シート: 希望_・スケジュール_ がある方）を開く
2. メニューから **「拡張機能」→「Apps Script」** をクリック
3. エディタで以下の3ファイルを作成:
   - `common.gs` — `gas/common.gs` の内容を貼り付け
   - `saturday.gs` — `gas/saturday.gs` の内容を貼り付け
   - `weekday.gs` — `gas/weekday.gs` の内容を貼り付け
4. `common.gs` の `MASTER_SPREADSHEET_ID` にマスタ用スプレッドシートのIDを設定（必須）
   ```javascript
   var MASTER_SPREADSHEET_ID = "ここにマスタ用スプレッドシートのIDを貼り付け";
   ```
   ※ スプレッドシートIDはURLの `https://docs.google.com/spreadsheets/d/【ここがID】/edit` の部分
5. `ADMIN_EMAIL` に管理者のメールアドレスを設定（全員完了通知・期限超過通知に必要）
   ```javascript
   var ADMIN_EMAIL = "admin1@example.com, admin2@example.com";
   ```
   ※ カンマ区切りで複数アドレスを指定可能
6. 本番運用時は `TEST_MODE` を `false` に変更
   ```javascript
   var TEST_MODE = false;
   ```
7. 上部の **「保存」**（💾アイコン）をクリック

## ステップ3: テスト実行

1. Apps Scriptエディタ上部の関数選択で **`testSendReminder`** を選択
2. **「実行」** をクリック
3. 初回はGoogleアカウントの **権限承認** を求められるので許可する
   - 「このアプリは確認されていません」→「詳細」→「（安全ではないページに）移動」
   - 許可する権限: Gmail送信、スプレッドシート読み取り
4. 下部の「実行ログ」で処理結果を確認

> 注意: テスト実行は翌日のスケジュールを参照する。翌日に確定済みの外勤がなければ何も送信されない。

## ステップ4: 定期実行トリガーを設定

Apps Scriptエディタ左側の **時計アイコン（トリガー）** から以下を追加:

### 土曜リマインダー（毎週金曜）

| 項目 | 設定値 |
|------|--------|
| 実行する関数 | `sendFridayReminder` |
| イベントのソース | 時間主導型 |
| タイプ | 週ベースのタイマー |
| 曜日 | 毎週金曜日 |
| 時刻 | 午後6時〜7時 |

### 土曜入力期限チェック（毎日）

| 項目 | 設定値 |
|------|--------|
| 実行する関数 | `checkDeadline` |
| イベントのソース | 時間主導型 |
| タイプ | 日付ベースのタイマー |
| 時刻 | 午前9時〜10時 |

### 平日入力期限チェック（毎日）

| 項目 | 設定値 |
|------|--------|
| 実行する関数 | `checkWeekdayDeadlines` |
| イベントのソース | 時間主導型 |
| タイプ | 日付ベースのタイマー |
| 時刻 | 午前9時〜10時 |

### 平日前日リマインダー（毎日）

| 項目 | 設定値 |
|------|--------|
| 実行する関数 | `sendWeekdayDayBeforeReminder` |
| イベントのソース | 時間主導型 |
| タイプ | 日付ベースのタイマー |
| 時刻 | 午後6時〜7時 |

## ステップ5: Web Appデプロイ（確定通知・希望入力通知用）

1. Apps Scriptエディタ右上の **「デプロイ」→「新しいデプロイ」** をクリック
2. 左上の歯車アイコンから **「ウェブアプリ」** を選択
3. 以下の通り設定する:

| 項目 | 設定値 |
|------|--------|
| 説明 | 外勤調整 通知 |
| 次のユーザーとして実行 | 自分 |
| アクセスできるユーザー | 全員 |

4. **「デプロイ」** をクリック
5. 表示された **ウェブアプリURL** をコピーする
6. StreamlitアプリのsecretsまたはStreamlit Cloudの設定に以下を追加:
   ```toml
   gas_webapp_url = "https://script.google.com/macros/s/XXXX.../exec"
   ```

## 平日セクション追加手順

新しい平日外勤セクションを追加する際、**GASの変更・再デプロイは不要**です。

1. 新しいGoogleスプレッドシートを作成
2. サービスアカウント（GASを設定したGoogleアカウント）に**編集者権限**で共有
3. Streamlitアプリの主管理者画面 → 平日外勤セクション管理で:
   - 新しいセクションを作成
   - **スプレッドシートキー**に作成したスプレッドシートのIDを入力
4. 完了（シートは自動作成されます）

## Streamlit secrets 設定項目一覧

```toml
# 必須
spreadsheet_key = "..."
spreadsheet_key_operational = "..."
[gcp_service_account]
# ...

# メール通知機能を使う場合に追加
gas_webapp_url = "https://script.google.com/macros/s/.../exec"
```

## トラブルシューティング

### リマインダーメールが届かない場合

1. Apps Scriptの「実行数」ページでエラーがないか確認
2. 確認ポイント:
   - 「スケジュールシートが見つかりません」→ 対象月のスケジュールが未作成
   - 「翌日の外勤割り当てはありません」→ 確定済みスケジュールに翌日の割り当てがない
   - 「メールアドレス未設定のためスキップ」→ 管理画面でメールアドレスを登録する
   - 「送信失敗」→ メールアドレスの形式を確認する

### 確定通知・希望入力通知メールが届かない場合

1. StreamlitのsecretsにGAS Web AppのURLが正しく設定されているか確認
2. Apps Scriptの「実行数」ページでエラーがないか確認
3. Web Appを再デプロイする（コード変更後は毎回再デプロイが必要）

### 平日セクションの通知が届かない場合

1. `平日外勤設定` シートに `spreadsheet_key` が正しく設定されているか確認
2. セクション別スプレッドシートがGASのGoogleアカウントに共有されているか確認
3. GASの実行ログで「スプレッドシートが未設定」エラーがないか確認

### 送信元アドレスを変更したい場合

送信元はスクリプトの権限を承認したGoogleアカウントに固定される。
変更するには、別のGoogleアカウントでスプレッドシートを開き、Apps Scriptの権限を再承認する。
