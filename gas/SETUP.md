# メールリマインダー・通知 セットアップ手順

## 機能概要

1. **毎週金曜18時のリマインダー**: 翌日（土曜）の外勤がある医員にメール送信
2. **スケジュール確定通知**: 管理者がスケジュールを確定したタイミングで全医員にメール送信
   - HTMLメールにスケジュール表画像をインライン埋め込み（`Charts.newTableChart`）
3. **Googleカレンダー連携**: スケジュール確定時に共有カレンダーへイベントを自動登録
4. **希望入力確認メール**: 医員が希望を保存するたびに本人へ入力内容の確認メールを送信
5. **全員入力完了通知**: 全医員の希望入力が完了した時点で管理者にメール送信
6. **入力期限リマインダー**: 期限日当日に全医員へ期限通知、翌日に管理者へ未入力者リストを送信

## 前提

- 外勤調整データのGoogleスプレッドシートが稼働中であること
- 管理画面で医員のメールアドレスが登録済みであること

## 仕組み

- Google Apps Script（スプレッドシート組み込み）を使用
- **送信元**: スクリプトを設定したGoogleアカウントのGmailアドレス
- **送信者名**: 「外勤調整システム」（スクリプト内の `SENDER_NAME` で変更可能）
- **追加費用**: なし

## ステップ1: 管理画面でメールアドレスを登録

1. Webアプリの管理者画面 → マスタ管理タブを開く
2. 各医員の「📧設定」ボタンからメールアドレスを登録する
3. メールアドレス未設定の医員にはリマインダー・通知は送信されない

## ステップ2: Google Apps Scriptを設定

1. **運用データ用スプレッドシート**（月別シート: 希望_・スケジュール_ がある方）を開く
2. メニューから **「拡張機能」→「Apps Script」** をクリック
3. エディタが開いたら、既存のコードをすべて削除
4. `gas/reminder.gs` の内容をすべてコピーして貼り付ける
5. `MASTER_SPREADSHEET_ID` にマスタ用スプレッドシートのIDを設定（必須）
   ```javascript
   var MASTER_SPREADSHEET_ID = "ここにマスタ用スプレッドシートのIDを貼り付け";
   ```
   ※ スプレッドシートIDはURLの `https://docs.google.com/spreadsheets/d/【ここがID】/edit` の部分
6. `ADMIN_EMAIL` に管理者のメールアドレスを設定（全員完了通知・期限超過通知に必要）
   ```javascript
   var ADMIN_EMAIL = "admin1@example.com, admin2@example.com";
   ```
   ※ カンマ区切りで複数アドレスを指定可能
   ※ 未設定の場合、管理者向け通知は送信されない（医員向け通知には影響なし）
7. **（任意）** Googleカレンダー連携を使用する場合、`CALENDAR_ID` を設定
   ```javascript
   var CALENDAR_ID = "abc123@group.calendar.google.com";
   ```
   ※ 共有カレンダーのIDは「Googleカレンダー設定 → カレンダーの統合 → カレンダーID」から取得
   ※ 空欄のままにするとカレンダー連携は無効（メール通知のみ動作）
8. 本番運用時は `TEST_MODE` を `false` に変更
   ```javascript
   var TEST_MODE = false;
   ```
9. 上部の **「保存」**（💾アイコン）をクリック

## ステップ3: テスト実行

### ドライラン（メール送信なし）

1. Apps Scriptエディタ上部の関数選択で **`dryRunReminder`** を選択
2. **「実行」** をクリック
3. 初回はGoogleアカウントの **権限承認** を求められるので許可する
   - 「このアプリは確認されていません」→「詳細」→「（安全ではないページに）移動」
   - 許可する権限: Gmail送信、スプレッドシート読み取り
4. 下部の「実行ログ」で医員名・外勤先・メールアドレスが正しく表示されるか確認

### 実メール送信テスト

1. 関数選択で **`testSendReminder`** を選択して「実行」
2. 実際にメールが届くことを確認する

> 注意: テスト実行は翌日のスケジュールを参照する。翌日に確定済みの外勤がなければ何も送信されない。

### スケジュール表画像テスト

1. 関数選択で **`testTableImage`** を選択して「実行」
2. ログに画像サイズとGoogle DriveのURLが出力される
3. URLを開いてテーブル画像が正しく生成されていることを確認

### カレンダー連携テスト（CALENDAR_ID 設定後）

1. 関数選択で **`testCalendarEvents`** を選択して「実行」
2. 初回は**カレンダーへのアクセス権限**の承認を求められるので許可する
3. 共有カレンダーにテストイベントが作成されることを確認

## ステップ4: 定期実行トリガーを設定

### リマインダー用（毎週金曜）

1. Apps Scriptエディタ左側の **時計アイコン（トリガー）** をクリック
2. 右下の **「＋トリガーを追加」** をクリック
3. 以下の通り設定する:

| 項目 | 設定値 |
|------|--------|
| 実行する関数 | `sendFridayReminder` |
| デプロイ | Head |
| イベントのソース | 時間主導型 |
| 時間ベースのトリガーのタイプ | 週ベースのタイマー |
| 曜日 | 毎週金曜日 |
| 時刻 | 午後6時〜7時 |
| エラー通知設定 | 毎日通知を受け取る（推奨） |

4. **「保存」** をクリック

### 入力期限チェック用（毎日）

1. **「＋トリガーを追加」** をクリック
2. 以下の通り設定する:

| 項目 | 設定値 |
|------|--------|
| 実行する関数 | `checkDeadline` |
| デプロイ | Head |
| イベントのソース | 時間主導型 |
| 時間ベースのトリガーのタイプ | 日付ベースのタイマー |
| 時刻 | 午前9時〜10時 |
| エラー通知設定 | 毎日通知を受け取る（推奨） |

3. **「保存」** をクリック

> 期限日はWebアプリの管理画面（マスタ管理 → 入力期限）で設定します。
> - **期限日当日の朝**: 全医員に「本日が入力期限です」と通知（入力済み/未入力の状況付き）
> - **期限日翌日の朝**: 管理者に未入力者リストを通知（全員入力済みなら通知なし）

## ステップ5: Web Appデプロイ（確定通知・希望入力通知用）

1. Apps Scriptエディタ右上の **「デプロイ」→「新しいデプロイ」** をクリック
2. 左上の歯車アイコンから **「ウェブアプリ」** を選択
3. 以下の通り設定する:

| 項目 | 設定値 |
|------|--------|
| 説明 | 外勤調整 通知 |
| 次のユーザーとして実行 | 自分 |
| アクセスできるユーザー | 全員 |

4. **「デプロイ」** をクリック
5. 表示された **ウェブアプリURL** をコピーする
6. StreamlitアプリのsecretsまたはStreamlit Cloudの設定に以下を追加:
   ```toml
   gas_webapp_url = "https://script.google.com/macros/s/XXXX.../exec"
   ```

## Streamlit secrets 設定項目一覧

```toml
# 必須
spreadsheet_key = "..."
spreadsheet_key_operational = "..."
[gcp_service_account]
# ...

# メール通知機能を使う場合に追加
gas_webapp_url = "https://script.google.com/macros/s/.../exec"
```

## Googleカレンダー連携の設定（任意）

共有カレンダーを使ってスケジュール確定時に自動でイベントを登録できます。

### 共有カレンダーの作成

1. [Googleカレンダー](https://calendar.google.com) を開く
2. 左側の「他のカレンダー」の **「＋」** → **「新しいカレンダーを作成」**
3. カレンダー名を入力（例: 「外勤スケジュール」）
4. **「カレンダーを作成」** をクリック
5. 作成したカレンダーの **「⋮」→「設定と共有」** を開く
6. **「特定のユーザーまたはグループと共有する」** で医員のメールアドレスを追加
   - 権限: 「予定の表示（すべての予定の詳細）」を推奨
7. **「カレンダーの統合」** セクションの **カレンダーID** をコピー
8. GASスクリプト内の `CALENDAR_ID` にペースト

### カレンダーイベントの仕様

| 項目 | 内容 |
|------|------|
| タイトル | 【外勤】{医員名} → {外勤先名} |
| 時間 | 外勤先のtime_slot設定に基づく（AM: 9-13時, PM: 13-17時） |
| 場所 | 外勤先マスタのlocationフィールド |
| 説明 | 医員名、外勤先名、場所 |

- スケジュールを**再確定**した場合、対象月の既存イベントは自動削除されてから再作成される
- `CALENDAR_ID` が空欄の場合、カレンダー連携は完全にスキップされる

## トラブルシューティング

### リマインダーメールが届かない場合

1. Apps Scriptエディタで `dryRunReminder` を実行し、ログを確認
2. 確認ポイント:
   - 「スケジュールシートが見つかりません」→ 対象月のスケジュールが未作成
   - 「翌日の外勤割り当てはありません」→ 確定済みスケジュールに翌日の割り当てがない
   - 「メールアドレス未設定のためスキップ」→ 管理画面でメールアドレスを登録する
   - 「送信失敗」→ メールアドレスの形式を確認する

### 確定通知・希望入力通知メールが届かない場合

1. StreamlitのsecretsにGAS Web AppのURLが正しく設定されているか確認
2. Apps Scriptの「実行数」ページでエラーがないか確認
3. Web Appを再デプロイする（コード変更後は毎回再デプロイが必要）

### 全員完了通知・期限超過通知が届かない場合

1. `ADMIN_EMAIL` が正しく設定されているか確認
2. GASのコードを更新した場合はWeb Appの**再デプロイ**が必要（「デプロイ」→「デプロイを管理」→鉛筆アイコン→「新しいバージョン」→「デプロイ」）
3. 期限超過通知は `checkDeadline` のトリガーが設定されているか確認
4. 管理画面で入力期限が設定されているか確認

### カレンダーにイベントが登録されない場合

1. `CALENDAR_ID` が正しく設定されているか確認
2. `testCalendarEvents` を実行してエラーがないか確認
3. カレンダーの権限を確認（スクリプト実行者がカレンダーに書き込み権限を持っている必要がある）
4. GASのコードを更新した場合はWeb Appの**再デプロイ**が必要

### 確定通知メールに画像が表示されない場合

1. Gmailの設定で「外部画像を表示する」が有効になっているか確認
2. GASの実行ログで「テーブル画像生成失敗」が出ていないか確認
3. 画像生成に失敗した場合はHTMLテーブルが自動的に代替表示される

### 送信元アドレスを変更したい場合

送信元はスクリプトの権限を承認したGoogleアカウントに固定される。
変更するには、別のGoogleアカウントでスプレッドシートを開き、Apps Scriptの権限を再承認する。
